# Twitch Local AI Translator Bot 開発要件定義書

## 1. プロジェクト概要
Twitchの配信チャットを監視し、海外視聴者からの外国語コメントを検知して、自動で日本語に翻訳してチャットに返信するボットを開発する。
外部API（DeepLやOpenAI等）を一切使用せず、**`Transformers.js` を用いたローカル環境での完全オフライン言語判定・翻訳**を実現することが本プロジェクトの最大の目的である。

## 2. 技術スタック
*   **言語:** TypeScript / Node.js
*   **Twitch連携:** `tmi.js`
*   **AI翻訳エンジン:** `Transformers.js`
*   **言語判定:** `franc`
*   **コード品質:** ESLint, Prettier (またはBiome)

## 3. コア機能要件と重要実装ルール（※過去のバグ対策含む）

### ① ローカルAI翻訳（モデル選定と量子化の必須指定）
*   翻訳精度の観点から、モデルは必ず **`Xenova/nllb-200-distilled-600M`** を使用すること。（※`opus-mt` 系は JS 環境で MarianTokenizer 非互換によるハルシネーションが発生するため絶対に使用しないこと）
*   **【超重要】** 初回ダウンロード時のタイムアウトとメモリ逼迫を防ぐため、`pipeline` のロード時には必ず **`{ quantized: true }`** を明示的に指定し、4-bit量子化モデルを使用すること。

### ② NLLB用 言語コードマッピング処理
*   NLLBモデルは BCP-47 形式の特殊な言語コードを要求する。
*   そのため、`franc` で判定した3文字の言語コード（`eng`, `spa`, `fra` 等）を、NLLBが認識できる形式（`eng_Latn`, `spa_Latn`, `fra_Latn` 等）にマッピング（変換）する関数を必ず間に挟むこと。
*   翻訳のターゲット言語（出力）は `jpn_Jpan` を指定すること。

### ③ 短文特有の言語誤判定の防止
*   Twitch特有の短いコメントが `fuf` や `sot` といったマイナー言語に誤判定されるのを防ぐため、`franc` のオプションで `minLength: 3` 程度に設定し、さらに `only` オプションを使用して判定候補を主要言語（`eng`, `spa`, `fra`, `jpn` 等）のみに制限すること。

### ④ パフォーマンス最適化（ショートカット辞書機能）
*   短い挨拶等で重いAI推論を走らせないため、AIに渡す前の事前フィルタとして「ショートカット辞書」を実装すること。
*   例: `hello` / `hi` -> 「こんにちは」、`gg` -> 「グッドゲーム！」、`lol` / `lmao` -> 「笑」
*   この判定は大文字小文字を区別せず、末尾の記号（! や ? など）をトリムしてから判定すること。

### ⑤ Twitchチャット送受信UX
*   ボット自身の発言や、「!」から始まるコマンドは処理をスキップする。
*   開発テスト用として `.env` に `PROCESS_OWN_MESSAGES=true` を設定した場合は、自分自身の発言も翻訳対象とすること。
*   翻訳結果の送信フォーマット: `[🇺🇸] @ユーザー名: 翻訳結果の日本語` （判定した言語コードに応じた国旗絵文字を付与する）

## 4. アーキテクチャとディレクトリ構成
関心の分離を意識し、以下のようにファイルを分割すること。
*   `src/index.ts` (エントリーポイント)
*   `src/twitch.ts` (tmi.jsの接続・送受信ロジック)
*   `src/translator.ts` (Transformers.jsのロードとNLLBマッピング・翻訳実行)
*   `src/language.ts` (francによる言語判定ロジック)
*   `src/shortcuts.ts` (ショートカット辞書の定義と判定処理)

## 5. AIへの実装ステップ指示
このドキュメントを読み込んだAIは、以下のステップに従って順番に実装を進めてください。

*   **Step 1:** プロジェクトの初期化（`package.json`, `tsconfig.json`, `ESLint/Prettier` のセットアップ）とパッケージのインストール。
*   **Step 2:** `.env.example` の作成と、`tmi.js` を用いたTwitchへの接続および受信ログ出力のベース実装（`src/twitch.ts`, `src/index.ts`）。
*   **Step 3:** ショートカット辞書機能（`src/shortcuts.ts`）と言語判定モジュール（`src/language.ts`）の実装。francの誤判定対策を必ず組み込むこと。
*   **Step 4:** `Transformers.js` を用いた翻訳モジュール（`src/translator.ts`）の実装。`quantized: true` の指定と BCP-47 コードへのマッピングを必ず行うこと。最後に全モジュールを結合し、Twitchへの国旗付き返信処理を完成させる。
